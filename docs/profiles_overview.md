# Profiles overview

> English is the primary text. The German clone is `profiles_overview_DE.md`.

The profile directory contains tool-specific configuration examples for working with local models.
The actual config files stay language-neutral.

## Included profiles

### `profiles/ollama/`
Short model notes and starter suggestions for local Ollama usage.

### `profiles/aider/`
Example Aider configuration for repo-focused terminal work with a local coding model.

### `profiles/continue/`
Example Continue configuration for editor-based local model usage.

## Why there are no `_DE` duplicates for config files

Config files are primarily technical and should stay shared to avoid drift between two versions.
Instead, the explanation layer is bilingual while the executable config stays single-source.
