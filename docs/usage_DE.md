# Nutzung

## Typische Einsätze

- ein Repo lokal prüfen
- Architekturdrift erkennen
- Umsetzungs-Prompts schreiben
- Coding-Hilfe ohne starke Cloud-Abhängigkeit nutzen

## Grundablauf

1. Stack starten.
2. Ein lokales Modell laden.
3. Open WebUI öffnen.
4. Prompts aus `prompts/` übernehmen oder anpassen.
5. Bei Bedarf Editor-Integrationen verwenden.

## Empfohlenes Muster

- Chat-Oberfläche für Planung und Audits
- Editor-Integration für Code-Änderungen
- Prompt-Bibliothek für wiederholbare Aufgaben

## Sprachregel

Englische Dateien sind Primärtexte.
Deutsche `_DE`-Dateien sind direkte Klone.
