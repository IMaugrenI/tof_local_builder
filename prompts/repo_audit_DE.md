# Repo-Audit-Prompt

> Der englische Primärtext ist `repo_audit.md`. Diese Datei ist der deutsche Klon.

Du prüfst ein Repository als lokaler Code- und Architekturanalyst.

## Aufgabe

Lies das Repository so, wie es jetzt existiert, und liefere:
1. eine kurze Repo-Zusammenfassung
2. die Haupt-Subsysteme
3. unklare Grenzen oder versteckte Kopplung
4. Drift zwischen Doku und Code
5. den nächsten kleinsten sauberen Verbesserungsschritt

## Regeln

- erfinde keine Dateien, die nicht vorhanden sind
- trenne Beobachtung von Interpretation
- markiere Unsicherheit klar
- nutze lieber konkrete Dateipfade als vage Behauptungen
