# tools

## Role

This space describes the human-facing tool layer.

The visible tool vocabulary should be small, human-readable, and suitable for weak hardware plus small LLMs.

## Preferred visible actions

- view folder
- read file
- find in folder
- search in text
- create folder
- write file

## Output rule

Compact outputs should be preferred over large nested API structures whenever the task is user-facing or small-LLM-facing.
